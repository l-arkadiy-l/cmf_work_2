# Task 2

Бейзлайн фильтр токсичных сделок для мейкера на Binance BTC/ETH. Фильтруем сделку (f=1),
если ждём от неё отрицательный maker PnL. Метрика Score(t) = PnL_kept минус PnL_all по
горизонтам 30/120/300 сек, оборот оставленных сделок держим не меньше 500к в день.
Маркаут и Score считаю ровно как в description.md.

Решил не делать как все (без логрега и без порога по знаковому ликвид-ноушналу). Собрал
простую таблицу средних. Бью сделки на ячейки по двум осям и смотрю средний PnL в каждой
ячейке на трейне. Первая ось это затухающее давление ликвидаций со стороны сделки
(экспонента, lambda 30 сек, Binance плюс Bybit со сдвигом 200мс), вторая это дисбаланс
очереди в стакане. Где средний PnL отрицательный, ту ячейку считаю токсичной и режу.
По сути обычный groupby, ничего не обучается.

Результаты на валидации (февраль):

* BTC: Score 0.60 / 0.23 / 0.19
* ETH: 1.94 / 2.47 / 0.98 (считал на сэмпле 1/20, оборот домножил на 20)

Оборот везде сильно выше 500к, ограничение не жмёт. Отфильтрованные сделки в минусе по PnL,
то есть режем именно токсик.

Файлы: task2_baseline.ipynb (весь код, графики, метрики, функция predict), cmf_lib.py
(загрузка через polars, маркаут, Score). Данные лежат в ../liquidation_task/data.
